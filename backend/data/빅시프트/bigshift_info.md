# 회사소개서: 빅시프트 (Bigshift)

## 1. 기업 개요
* **슬로건:** AI 대전환을 위한 개발 및 컨설팅 파트너
* **핵심 가치:** AI 기획 컨설팅부터 빠르고 안정적인 배포까지 모든 과정 수행
* **설립일:** 2025년 9월
* **실적:** 설립 직후 10건 이상의 AI 프로젝트 성공
* **주요 고객 및 프로젝트:**
    * 문화체육관광부 (AI 기반 업무 자동화)
    * SNUH 서울대학교병원 (자동 이상 감지, AI 레포트, 챗봇 등)
    * 기타 다양한 스타트업 및 중견기업 (마케팅 소재 생성, 비정형 데이터 분석, 사내 RAG 등)


## 2. 시장의 문제점 (Why AI Projects Fail)
**미국 RAND 연구소 보고:** AI 프로젝트 실패율은 일반 IT 프로젝트의 2배

### 문제 1: PM(Project Manager)의 역량 부족
* **현황:** 기존 개발사의 비개발자 출신 PM은 급변하는 AI 기술을 따라가지 못함.
* **결과:** 구현 단계에서 기술적 한계 봉착.
* **실패 사례 (Case 1):**
    * *기획:* 광고 이미지 리사이징 자동화 요청.
    * *현실:* OpenAI는 비트맵을 매번 새로 생성하므로 텍스트 재배치 연산이 과다하고 변형이 심함. 단순 코드로 구현하는 것과 AI 활용의 차이를 이해하지 못해 범용성이 낮은 결과물 초래.

### 문제 2: 개발자의 역량 부족
* **현황:** 익숙한 스택만 고집하며 최신 AI 아키텍처(파편화된 기술) 습득 부족.
* **결과:** 낮은 성능과 확장성 한계.
* **실패 사례 (Case 2 - RAG 챗봇):**
    * *기획:* 이미지 위주의 사내 문서를 텍스트 추출로만 처리하고 구형 임베딩 사용.
    * *현실:* 이미지 내 핵심 내용 소실. OpenAI 구형 Embedding 모델은 최신 상위 스택 대비 정확도가 절반 이하로 낮음.


## 3. 빅시프트의 솔루션 및 강점
### 3.1. 조직 구조 혁신 (AI 프로젝트 특화)
* **기존 개발사:** PM, 디자이너, 백엔드, 프론트엔드가 분절되어 복잡한 의사소통 필요.
* **빅시프트:** **전 구성원이 풀스택(Full-stack) 개발자이자 PM** 역할 수행.
    * 각 기능을 분담하여 최적의 의사결정을 빠르게 내림.
    * 국내 유일 전 구성원 풀스택 개발/기획/출시 경험 보유 팀.

### 3.2. 인재 밀도
* **구성원 배경:** 유니콘 스타트업(오늘의집, ABLY 등) 주축 멤버 및 명문대(서울대, 고려대, POSTECH 등) 출신.
* **역량:** 높은 Raw Intelligence를 기반으로 새로운 AI 기술을 기민하게 흡수.

### 3.3. 성과 데이터
* **고객 문의 25% 감소:** 중견기업 A사 (ERP RAG 챗봇 도입)
* **사용자 5만 명 증가:** 스타트업 B사 (보고서 작성 AI, 6주 만에 달성 및 Seed 투자 유치)
* **제작 기간 95% 단축:** 스타트업 C사 (광고 소재 제작 AI)


## 4. 서비스 제공 방식 (Customized Development)
### Type A: 속도가 중요한 프로젝트
* **특징:** 마감 기한 엄수, 빠른 PoC(개념 증명)
* **핵심 기술:**
    * **SOTA 모델 적용:** 최신 원천 모델 활용.
    * **성능 최적화:** Prompt 엔지니어링, RAG 최적화, 벡터 DB 튜닝 내재화.
    * **API 튜닝:** 비용 및 응답 품질 분석.
* **효과:** 사내 RAG 서비스 프로젝트 기간 타사 대비 1/2 단축.

### Type B: 보안이 중요한 프로젝트 (의료/공공기관)
* **특징:** 데이터 유출 방지, 폐쇄망 환경
* **핵심 기술:**
    * **자체 모델(On-premise/Local LLM) 구축:** 자체 GPU 서버 활용.
    * **보안 인프라:** 망분리, 접근통제, 암호화, 인증 체계(ISMS-P 등) 준수.
    * **문서화:** 정부 기관 납품용 기술 문서 제공.

### 1) 대형병원 ERP 매뉴얼 RAG 챗봇
* **고객사:** 중대형병원 전용 ERP 제공 업체
* **목표:** 매뉴얼 기반 정확한 응답을 통한 고객센터 인입 감소.
* **기술적 강점:**
    * 데이터 업로드 시 AI 자동 반영 파이프라인.
    * 출처 페이지(스크린샷 포함)를 이미지로 함께 제시하여 신뢰도 확보.
    * 이미지 내 버튼/UI를 텍스트로 전처리 변환.
    * 불분명한 질문을 구체화하는 역질문(Re-asking) AI 레이어 구축.

### 2) 보고서 작성 AI 서비스
* **고객사:** 교육 스타트업
* **목표:** 중/고등학생 수행평가 보고서 작성 보조.
* **기술적 강점:**
    * 장문 작성을 위한 고도화된 AI 체이닝(Chaining).
    * 교과서 RAG + 외부 논문 Web-search AI Agent 결합.
    * 도표와 이미지가 포함된 보고서 생성 구현.
    * 일관된 톤앤매너 통제 기술.

### 3) 공무원 내규집 Q&A AI 챗봇
* **고객사:** 문화체육관광부 산하 기관
* **목표:** 방대한 내규집 임베딩을 통한 질의응답 챗봇.
* **기술적 강점:**
    * 1달 내 챗봇 및 관리 대시보드 풀스택 구현.
    * 복잡한 표, 부칙, 주석 등 비정형 정부 문서의 성공적 임베딩.
    * 기존 사내 웹페이지 이식 및 유지관리 비용 최소화 구조.

### 4) 광고 소재 이미지 제작 AI
* **고객사:** 광고대행사
* **목표:** 타사 레퍼런스 광고의 구도를 유지하되 브랜드에 맞춰 재생성.
* **기술적 강점:**
    * **한글 텍스트 문제 해결:** Vision AI로 텍스트 영역 인식 -> 폰트/색상 분석 -> 텍스트 교체 방식 구현.
    * 이미지와 텍스트 요소별 수정 가능한 시스템.
    * 톤앤매너 프롬프팅 체인 최적화.

### 5) 공연 예매 정보 AI 크롤링 시스템
* **고객사:** 티켓팅 플랫폼 스타트업
* **목표:** 비정형 예매 공지(YES24 등)를 정형 데이터로 변환.
* **기술적 강점:**
    * 비표준 문장 패턴을 정규화하는 AI 파싱 모델.
    * 대용량 비동기 데이터 파이프라인.
    * 중복 공고 및 선예매 공고 클러스터링.

### 6) 맞춤형 환자 관리 LLM 서비스
* **고객사:** 서울대학교 병원 산하 연구조직
* **목표:** 특정 질병 환자 대상 식단/복약/상담 관리 앱.
* **기술적 강점:**
    * **No API, Local LLM:** 자체 클라우드 서버에 모델을 올려 의료 데이터 보안 준수.
    * 제각각인 질병 문서를 RAG화하여 데이터 기반 응답.
    * 유저 기록(PHR) 반영 맞춤형 응답 제공.
