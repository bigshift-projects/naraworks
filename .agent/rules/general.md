# AI 에이전트 개발 핵심 원칙

## 프로젝트 철학
- **근본 원인 우선**: 문제 해결 시 중간 해결책보다 근본 원인 파악을 우선시
- **API First**: 모든 기능은 API 설계부터 시작
- **단순함 추구**: 복잡한 해결책보다 단순하고 명확한 접근 선호

## 개발 우선순위
1. **데이터 모델링**: 모든 기능의 기초
2. **API 설계**: 데이터 구조를 반영한 인터페이스
3. **비즈니스 로직**: 백엔드에서 핵심 로직 구현
4. **UI 구현**: 백엔드가 준비된 후 표현 계층 구현

## 품질 기준
- 모든 코드는 명확하고 자명해야 함
- 주석보다는 코드 자체가 의도를 설명하도록 작성
- 주석은 한글로 작성
- 일관된 패턴과 관례 유지
- 성능보다는 가독성과 유지보수성 우선

## 중요 참고사항
- 프로젝트의 기존 패턴과 일관성 유지
- 새로운 의존성 추가 시 신중하게 검토
- 복잡한 비즈니스 로직은 항상 백엔드에서 처리
- **⚠️ Rule 정확성 검증: Rule에 명시된 정보와 실제 프로젝트 상태가 불일치할 경우, 즉시 사용자에게 메인터넌스 필요성을 알리고 구체적인 불일치 내용을 보고할 것**
